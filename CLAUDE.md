# 会議ファシリテーター プロトタイプ — Claude Code 指示書

## 0. まず plan mode で

いきなり実装に入らないでください。まず **plan mode** で、この文書をもとにした全体設計・状態機械・ファイル構成・各画面のラフを提案してください。こちらが承認したら実装に進みます。

## 1. 何を作るか

会議の「目的」と「アジェンダ」を最初に読み込ませると、会議中ずっと音声を聞き取り、必要なときだけ **礼儀正しく挙手 → 議長の許可 → 音声で一言** という形で進行を補助し、終了後に議事録を生成するツール。

成果物は **HTML単体プロトタイプ**（1フォルダ・ローカルで動く）。デプロイはしない。

性格づけ: 受け身の進行ツールではなく「出しゃばらない同席型のAIファシリテーター」。**黙っているのがデフォルト**で、要所でだけ手を挙げる。

## 2. 動作の中心：常時ループ

会議中、次の4段階をずっと回し続ける。

```
聞く → 考える → 挙手して許可を待つ → 発話 → （聞くに戻る）
```

- **聞く**: 音声を常時文字起こしし、バッファに溜める。
- **考える**: 一定のトリガーで Claude に「介入すべきか」を判定させる。トリガーは「発言の区切り（無音 ≧ 約1.5秒）」＋「20秒ごとの定期チェック」＋「タイマーイベント（項目が時間超過など）」。
- **挙手して許可を待つ**: 介入したいときだけ、画面に挙手インジケータを出し、小さな合図音を鳴らす。議長が許可ボタンをタップするまで発話しない。
- **発話**: 許可されたら TTS で短く一言だけ話す（部屋全体に流す＝全参加者に聞こえる）。

### 分岐（重要）

- 「考える」で介入不要（`wants_to_speak: false`）なら、挙手せずそのまま聞くに戻る。
- 挙手しても議長が許可しない場合：無視・却下、または一定時間（TTL、例 90秒）で内容が古くなったら、**静かに取り下げて**聞くに戻る。古い指摘を蒸し返さない。

## 3. 並行処理（議事録ネタの蓄積）

上のループとは**別系統**で、低頻度（1分ごと、または項目切替時）に Claude を呼び、決定事項・アクション・未解決の論点を抽出してパネルに表示する。終了ボタンで全体から議事録を生成する。介入判定とは必ず別の呼び出しに分けること（各プロンプトを単機能に保つため）。

## 4. 機能（フェーズ分け）

### Phase 1（まず動かす）

- 会議の目的＋アジェンダのアップロード or 貼り付け → Claude で構造化（項目・時間配分）。
- 日本語のリアルタイム文字起こし（Web Speech API, Chrome 前提）。
- アジェンダ追従（「今ここ」表示）＋項目ごとのタイマー。
- 介入判定ループ（挙手UI＋合図音＋議長の許可ボタン＋TTS発話、部屋全体に流す）。
- 決定事項・アクション・論点の抽出パネル。
- 終了 → 議事録生成（画面表示＋コピー／ダウンロード）。
- 話者タグ付け: **手動**（議長が「現在の話者」をタップ）。データモデルに `speaker` フィールドを持たせる。1人運用・タグなしでも成立すること。

### Phase 2（後で・今回は実装しないが設計で塞がない）

- 話者の自動判別: diarization 対応のクラウドSTT（例: Deepgram / AssemblyAI）に差し替え、匿名の話者ラベル（Speaker 0/1/2…）を取得。
- 会議冒頭の自己紹介「私の名前は〇〇です」を Claude に解析させ、`話者ラベル → 実名` を自動マッピング。
- Next.js ＋ サーバーサイドプロキシで API キーを保護（デプロイ時）。

→ STT 層は差し替え可能なモジュールとして分離しておくこと。Phase 1 の Web Speech API と Phase 2 のクラウドSTTを、同じインターフェース（文字確定チャンク＋任意の `speaker` ラベルを emit する）の裏で入れ替えられる構造にする。

## 5. 技術構成（Phase 1）

| 役割 | 使うもの | 備考 |
|---|---|---|
| 音声 → 文字 | ブラウザ Web Speech API (`SpeechRecognition`, `lang='ja-JP'`, `continuous=true`, `interimResults=true`) | 無料・リアルタイム。Chrome/Edge 前提 |
| 思考（判定・抽出・議事録・アジェンダ解析） | Claude API (`/v1/messages`) | すべて構造化JSON／Markdownで返させる |
| 文字 → 音声 | Web Speech API (`SpeechSynthesis`, `lang='ja-JP'`) | プロトタイプには十分 |

## 6. 重要な割り切り・制約

- **ブラウザは Chrome/Edge 前提**（SpeechRecognition が Safari/Firefox で弱い）。起動時に未対応ブラウザを検知して警告を出す。
- **HTML単体プロトは API キーがクライアント側に露出する**。よって **ローカルでの個人検証専用**。公開・デプロイしないこと。本番化は Phase 2 のサーバーサイドプロキシで行う。コード内・コメントにこの注意を明記する。
- **単一マイク**前提。Phase 1 は話者分離なし（手動タグ）。
- 実会議の音声が外部API（Claude／将来のSTT）に送られる点に留意。検証はダミー会議で行う想定。本番化時に ZDR／Enterprise を再検討する。

## 7. Claude 呼び出し（4種・それぞれ単機能）

すべて「**JSON以外を一切出力するな（前置き・コードフェンス禁止）**」と明示し、パース前に ```json フェンスを除去して安全にパースすること。

### 7-1. アジェンダ解析（会議開始時に1回）

- 入力: アップロード／貼り付けされた目的＋アジェンダのテキスト
- 出力:
```json
{
  "purpose": "会議の目的（1〜2文）",
  "items": [
    { "id": "1", "title": "項目名", "allotted_minutes": 15 }
  ]
}
```

### 7-2. 介入判定（常時・発言区切り＋20秒ごと＋タイマーイベント）

- 入力: 会議の目的／全項目と現在地／経過・残り時間／直近数分の文字起こし／これまでに発した介入の履歴
- 出力:
```json
{
  "wants_to_speak": true,
  "type": "offtopic | time_over | agenda_progress | none",
  "urgency": "low | medium | high",
  "spoken_message": "TTSで読む短い一言（敬語・15秒以内で読める長さ）",
  "note": "記録用の内部メモ（読み上げない）"
}
```
- 方針: デフォルトは `false`（黙る）。同じ趣旨の介入を繰り返さない（履歴を見て判断）。`spoken_message` は敬語・簡潔・断定しすぎない。

### 7-3. 抽出（低頻度・1分ごと or 項目切替時）

- 入力: 直近の文字起こし＋現在の項目
- 出力:
```json
{
  "decisions": ["決まったこと"],
  "actions": [{ "who": "担当", "what": "やること", "due": "期限 or null" }],
  "open_questions": ["未解決の論点"]
}
```

### 7-4. 議事録生成（終了時に1回）

- 入力: 全文字起こし＋抽出済みの decisions / actions / open_questions ＋アジェンダ
- 出力: Markdown の議事録（目的・各アジェンダ項目の要約・決定事項・アクション一覧・残課題）

## 8. 状態機械（目安）

```
IDLE
  → SETUP        （目的・アジェンダ入力、参加者の自己紹介で名簿作成）
  → RUNNING
       ├ LISTENING     （文字起こし継続）
       ├ EVALUATING    （Claude 7-2 を呼ぶ）
       ├ HAND_RAISED   （挙手・許可待ち。TTLあり）
       └ SPEAKING      （TTS発話 → LISTENINGに戻る）
  → ENDED        （議事録生成・表示）
```
抽出（7-3）は RUNNING 中ずっとバックグラウンドで回る別タイマー。

## 9. UI構成（ラフ）

- **セットアップ画面**: 目的＋アジェンダの入力／貼り付け。参加者登録（各自「私の名前は〇〇です」と発言 → Phase 1 では名簿として手入力 or 自己紹介から拾う）。
- **進行画面**: 現在の項目＋残り時間タイマー／文字起こしのライブ表示／抽出パネル（決定・アクション・論点）／AIの挙手インジケータ＋「許可」ボタン／「現在の話者」をタップで選ぶ行。
- **終了画面**: 議事録表示＋コピー／ダウンロードボタン。

## 10. ファイル構成（提案）

1フォルダにまとめる。可能なら以下に分割（単一 index.html に内包しても可）:

- `index.html` — 画面・レイアウト
- `app.js` — 状態機械とメインループ
- `claude.js` — Claude API 呼び出し4種
- `speech.js` — STT/TTS ラッパー（Phase 2 で差し替え可能な形に）
- `agenda.js` — アジェンダ・名簿・抽出データの保持
